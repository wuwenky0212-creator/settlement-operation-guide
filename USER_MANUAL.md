## 系统概述

### 主要功能

Settlement Operation Guide提供以下核心功能：

- **交易查询**: 支持多条件组合查询交易记录
- **交易详情**: 查看交易的完整信息，包括事件、账务、支付信息
- **生命周期跟踪**: 实时跟踪交易处理进度
- **操作指引**: 根据当前状态提供下一步操作建议
- **现金流管理**: 查询和跟踪现金流收付进度
- **数据导出**: 支持Excel和CSV格式导出

### 系统界面

系统采用murex风格的界面设计，主要包括：

- **顶部导航栏**: 快速访问主要功能
- **查询条件区**: 输入查询条件
- **数据列表区**: 显示查询结果
- **详情弹窗**: 查看详细信息
- **操作按钮**: 执行各种操作

## 功能详解

### 1. 交易汇总查询

#### 功能说明

交易汇总页面允许您查询和浏览所有交易记录。

#### 使用步骤

1. **输入查询条件**
   - 在页面顶部的查询条件区域输入条件
   - 支持的查询条件包括：
     - 外部流水号（必填）
     - 交易状态
     - 交易日期范围
     - 起息日期范围
     - 到期日期范围
     - 交易对手
     - 产品类型
     - 货币
     - 运营机构
     - 业务机构
     - 清算方式
     - 证实方式
     - 交易来源

2. **执行查询**
   - 点击"查询"按钮或按Enter键
   - 系统将在下方表格中显示查询结果

3. **浏览结果**
   - 查询结果以表格形式展示
   - 默认按交易日期降序排列
   - 显示关键信息：流水号、交易日、产品、状态等

#### 查询技巧

- **精确查询**: 使用外部流水号可以快速定位特定交易
- **范围查询**: 使用日期范围可以查询一段时间内的交易
- **组合查询**: 可以同时使用多个条件缩小查询范围

#### 示例

**查询2026年2月的所有外汇即期交易**:
```
交易日起始: 2026-02-01
交易日结束: 2026-02-28
产品: 外汇即期
```

---

### 2. 分页浏览

#### 功能说明

当查询结果较多时，系统会自动分页显示。

#### 使用方法

1. **选择每页显示数量**
   - 在左下角选择：15/20/50/100条
   - 系统会记住您的选择

2. **翻页**
   - 点击"首页"跳转到第一页
   - 点击"上一页"查看前一页
   - 点击"下一页"查看后一页
   - 点击"末页"跳转到最后一页
   - 直接输入页码跳转到指定页

3. **查看统计信息**
   - 右下角显示"显示X到Y，共Z记录"
   - 帮助您了解当前位置和总记录数

---

### 3. 交易详情查看

#### 功能说明

点击交易记录可以查看该交易的完整详细信息。

#### 使用步骤

1. **打开详情**
   - 在交易列表中点击任意交易记录
   - 系统会打开交易详情弹窗

2. **浏览标签页**
   - 详情页包含6个标签页：
     - **交易信息**: 交易基本信息、账户信息、备注
     - **价格审核**: 价格审核相关信息
     - **授信信息**: 授信额度和使用情况
     - **支付信息**: 清算方式和支付详情
     - **事件信息**: 事件流转记录
     - **账务信息**: 账务处理记录

3. **切换标签页**
   - 点击标签页标题切换
   - 选中的标签页显示为红色

4. **关闭详情**
   - 点击右上角的"X"按钮
   - 或按Esc键关闭

#### 交易信息标签页

显示交易的核心信息：

- **货币对**: 如USD/CNY
- **金额**: BUY金额和SELL金额（带方向标识）
- **汇率**: 即期汇率、点差、成本汇率
- **日期**: 交易日、交易时间、起息日
- **账户**: 交易账户、交易对手、经纪商
- **备注**: 交易性质、外部流水号、交易目的

#### 事件信息标签页

显示交易的所有事件记录：

- 按时间倒序排列（最新的在前）
- 包含14个字段：
  - 外部流水号
  - 交易流水号
  - 父交易流水号
  - 产品
  - 账户
  - 事件类型
  - 交易状态
  - 录入日
  - 交易日
  - 修改日
  - 后线处理状态
  - 证实状态
  - 证实匹配状态
  - 操作用户

#### 支付信息标签页

显示清算和支付信息：

- **清算方式**:
  - 我行开户行、账号、户名
  - 对手开户行、账号、户名
- **支付信息**:
  - 指令ID
  - 支付日期
  - 报文类型
  - 币种和金额
  - 报文发送人和时间

#### 账务信息标签页

显示账务处理记录：

- 传票号
- 实际记账日
- 计划记账日
- 事件号
- 借贷方向
- 货币
- 科目
- 交易金额

底部显示按币种汇总的借贷金额。

### 4. 生命周期进度跟踪

#### 功能说明

实时跟踪交易在结算流程中的处理进度。

#### 查看进度

在交易详情页面，系统会显示：

1. **当前阶段**: 交易所处的具体阶段
2. **进度百分比**: 整体完成进度
3. **流程可视化**: 图形化展示流程

#### 流程阶段

交易处理包含以下主要阶段：

1. **SWIFT证实**
   - 发送RMC
   - 由RMC发送FTM

2. **证实匹配**
   - 系统自动匹配
   - 匹配成功/失败

3. **收付发报**
   - 反洗钱检查
   - SWIFT发报
   - 核心入账

#### 状态标识

- **绿色**: 已完成
- **蓝色**: 当前处理中
- **灰色**: 未开始
- **红色**: 失败或阻塞

#### 实时更新

- 系统每10秒自动更新状态
- 您也可以点击"刷新"按钮手动更新

### 5. 操作指引

#### 功能说明

根据交易当前状态，系统会自动生成操作指引，告诉您下一步应该做什么。

#### 查看指引

在交易详情页面或进度跟踪页面，您会看到：

- **下一步操作**: 明确的操作说明
- **操作入口**: 可点击的按钮或链接
- **注意事项**: 需要注意的事项
- **预计时间**: 操作预计需要的时间

#### 常见操作指引

**场景1: 证实匹配失败**
```
下一步操作: 联系交易对手方核对交易信息，进行人工匹配
操作入口: [人工匹配处理] 按钮
注意事项: 请仔细核对交易金额、日期等关键信息
```

**场景2: 反洗钱检查等待中**
```
下一步操作: 等待反洗钱系统审核完成
操作入口: [反洗钱系统查询] 链接
预计时间: 通常在30分钟内完成
```

**场景3: 核心入账失败**
```
下一步操作: 查询内部账划转情况，联系核心系统支持
操作入口: [内部账划转查询] 按钮
注意事项: 请记录错误信息，便于技术支持排查
```

#### 执行操作

1. 阅读操作指引
2. 点击操作入口按钮或链接
3. 按照提示完成操作
4. 返回查看更新后的状态

### 6. 现金流查询

#### 功能说明

查询和管理交易相关的现金流记录。

#### 使用步骤

1. **进入现金流页面**
   - 点击顶部导航栏的"现金流查询"

2. **输入查询条件**
   - 交易流水号
   - 现金流内部ID
   - 收付信息ID
   - 结算内部ID
   - 方向（收/付）
   - 币种
   - 金额范围
   - 收付日期范围
   - 状态

3. **查看结果**
   - 现金流列表显示：
     - 现金流ID
     - 交易流水号
     - 方向
     - 币种和金额
     - 收付日期
     - 账号信息
     - 当前状态
     - 进度指示

4. **查看金额汇总**
   - 页面底部显示按币种和方向汇总的金额

---

### 7. 现金流收付进度

#### 功能说明

跟踪单个现金流的收付处理进度。

#### 查看进度

1. 在现金流列表中点击记录
2. 系统显示收付进度详情

#### 进度阶段

根据结算方式不同，显示不同的流程：

**SWIFT/HKSWIFT方式**:
1. 反洗钱检查
2. SWIFT发报
3. 核心入账

**内部账方式**:
1. 发送核心
2. 核心入账

#### 操作指引

系统会根据当前状态提供操作指引，使用方法与交易操作指引相同。

### 8. 数据导出

#### 功能说明

将查询结果导出为Excel或CSV文件，便于离线分析。

#### 使用步骤

1. **执行查询**
   - 先使用查询条件获取需要导出的数据

2. **点击导出按钮**
   - 在查询结果页面点击"导出"按钮

3. **选择导出选项**
   - 选择导出格式（Excel/CSV）
   - 选择要导出的字段（可选）

4. **确认导出**
   - 点击"确认"按钮
   - 系统开始生成文件

5. **下载文件**
   - 文件生成后自动下载
   - 文件名格式：`交易汇总_YYYYMMDD_HHMMSS.xlsx`

#### 导出限制

- 单次最多导出10,000条记录
- 如果超过限制，请缩小查询范围或分批导出

#### 导出技巧

**分批导出大量数据**:
```
1. 按月份分批查询和导出
2. 使用日期范围：2026-01-01 到 2026-01-31
3. 导出第一批
4. 继续下一个月：2026-02-01 到 2026-02-28
5. 重复直到完成
```

## 常见问题

### Q1: 为什么查询不到交易记录？

**可能原因**:
1. 外部流水号输入错误
2. 交易尚未录入系统
3. 查询条件过于严格

**解决方法**:
1. 检查外部流水号是否正确
2. 确认交易是否已经录入
3. 放宽查询条件，只使用必要的条件

---

### Q2: 交易状态长时间不更新怎么办？

**可能原因**:
1. 系统正在处理中
2. 遇到异常需要人工处理
3. 外部系统响应慢

**解决方法**:
1. 查看操作指引了解当前状态
2. 如提示需要人工处理，按指引操作
3. 如长时间无变化，联系技术支持

---

### Q3: 导出功能提示记录数超限怎么办？

**解决方法**:
1. 缩小查询范围（使用日期范围）
2. 分批导出
3. 联系管理员调整导出限制

---

### Q4: 如何查看历史操作记录？

**解决方法**:
1. 打开交易详情
2. 切换到"事件信息"标签页
3. 查看所有事件记录
4. 事件记录包含操作人和操作时间

---

### Q5: 证实匹配失败如何处理？

**解决方法**:
1. 查看操作指引
2. 点击"查看匹配差异"
3. 核对交易信息
4. 联系交易对手方确认
5. 使用"人工匹配处理"功能
6. 提交人工匹配请求

---

### Q6: 如何知道交易是否已完成？

**判断方法**:
1. 查看生命周期进度
2. 进度百分比达到100%
3. 所有阶段显示为绿色（已完成）
4. 操作指引显示"流程已完成，无需操作"

---

### Q7: 系统响应慢怎么办？

**可能原因**:
1. 查询数据量过大
2. 网络连接问题
3. 系统负载高

**解决方法**:
1. 缩小查询范围
2. 检查网络连接
3. 避开高峰时段
4. 联系技术支持

---

### Q8: 如何查看账务金额是否平衡？

**解决方法**:
1. 打开交易详情
2. 切换到"账务信息"标签页
3. 查看底部的金额汇总
4. 检查每个币种的借贷金额
5. 正常情况下，借贷金额应该平衡

## 技巧与最佳实践

### 查询技巧

1. **使用外部流水号快速定位**
   - 外部流水号是最精确的查询条件
   - 适合查询特定交易

2. **使用日期范围批量查询**
   - 适合查询一段时间内的交易
   - 建议不超过3个月

3. **组合多个条件精确查询**
   - 可以同时使用多个条件
   - 条件越多，结果越精确

4. **保存常用查询条件**
   - 浏览器会记住您的输入
   - 下次访问时自动填充

### 监控技巧

1. **定期检查待处理交易**
   - 每天查询一次当天的交易
   - 关注状态为"处理中"的交易

2. **关注失败和阻塞的交易**
   - 优先处理红色标记的交易
   - 及时查看操作指引

3. **使用轮询监控重要交易**
   - 保持详情页面打开
   - 系统会自动更新状态

### 操作技巧

1. **及时响应操作指引**
   - 系统提示需要人工处理时，尽快处理
   - 避免交易长时间阻塞

2. **记录重要信息**
   - 记录交易流水号
   - 记录错误信息
   - 便于后续追踪和排查

3. **使用快捷键**
   - Enter键：提交查询
   - Esc键：关闭弹窗
   - Tab键：在输入框间切换

### 导出技巧

1. **选择合适的格式**
   - Excel：适合复杂分析和格式化
   - CSV：适合大量数据和导入其他系统

2. **只导出需要的字段**
   - 减少文件大小
   - 提高导出速度

3. **分批导出大量数据**
   - 按月份或按产品类型分批
   - 避免单次导出过多数据

### 性能优化

1. **避免查询过大范围**
   - 不要查询超过6个月的数据
   - 使用更多条件缩小范围

2. **合理使用分页**
   - 不要设置过大的每页记录数
   - 推荐使用20或50条

3. **避开高峰时段**
   - 上午9-10点和下午2-3点是高峰
   - 尽量在其他时段操作

### 安全建议

1. **及时退出系统**
   - 离开时记得退出登录
   - 避免他人使用您的账号

2. **不要分享账号**
   - 每个人使用自己的账号
   - 便于追踪操作记录

3. **定期修改密码**
   - 建议每3个月修改一次
   - 使用强密码
